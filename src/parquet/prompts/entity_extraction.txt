-Goal-
Given a text document and a list of entity types, identify all entities
and the relationships between them. Output both entities and relationships
in a single pass using the specified formats.

-Steps-
1. Identify Entities: Extract entities of the following types: `[{entity_types}]`.
   For each entity, use this format:
   `("entity"{tuple_delimiter}<name>{tuple_delimiter}<type>{tuple_delimiter}<description>)`
   * Email:        `Subject: <subj> | Tone: <tone> | Attachments: <files/None> | Summary: <1-sentence>`
   * Person:       `Name: <name> | Account: <email/Unknown>`
   * Topic:        `Priority: <high/mid/low> | Summary: <1-sentence>`
   * Event:        `Schedule: <YYYY-MM-DD HH:MM> | Location: <place> | Status: <confirmed/tentative>`
   * Organization: `Context: <dept> | Team: <team>`

2. Identify Relationships: Based on the identified entities, extract clearly supported
   relationships using these types:
   `[SENT_BY, SENT_TO, CC_TO, DISCUSSED, BELONGS_TO, SCHEDULED_FOR, WAS_ATTACHED_TO]`
   Use this format:
   `("relationship"{tuple_delimiter}<source>{tuple_delimiter}<target>{tuple_delimiter}<type>{tuple_delimiter}<description_in_Korean>{tuple_delimiter}<strength_1_10>)`

   Relationship directions are fixed as follows — do not reverse them:
   | Type             | Source        | Target        |
   |------------------|---------------|---------------|
   | SENT_BY          | Email         | Person        |
   | SENT_TO          | Email         | Person        |
   | CC_TO            | Email         | Person        |
   | DISCUSSED        | Email         | Topic         |
   | BELONGS_TO       | Person        | Organization  |
   | SCHEDULED_FOR    | Email         | Event         |
   | WAS_ATTACHED_TO  | Topic/File    | Email         |

3. Constraints:
   * If the same entity appears multiple times in the text, extract it only once.
   * If the sender is unknown, use `Account: Unknown`.
   * Keep descriptions concise.
   * Relationship descriptions must be written in Korean.
   * Output as a single list delimited by `{record_delimiter}`.
   * End with `{completion_delimiter}`.

-Examples-
Example 1
Entity_types: Email, Person, Organization, Topic
