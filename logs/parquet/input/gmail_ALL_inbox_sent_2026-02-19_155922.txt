Build GraphRAG for Accurate, Contextual GenAI
Hey 인선,

Previous emails have explained what GraphRAG is and shown it in action. Now, you can learn step by step how to create a GraphRAG architecture with a knowledge graph.

A knowledge graph provides a mental model of your data with its connected entities and relationships. Using one, your LLM can access information about dependencies, sequences, hierarchies, and deeper, true meaning to provide the context GenAI accuracy needs.

Read this developer’s guide to GraphRAG <https://message.neo4j.com/NzEwLVJSQy0zMzUAAAGf5OUe3XFoX_2xqpqVhsOwGFvVsNOpPrdyVSmWWo_Ot3Cw-lbsh8QmEOG2jnPdpYK8lT8Hrz4=> to learn how to:

Hello 인선,

 We’re excited to introduce Neo4j Aura Agent, a new platform for building AI 
agents grounded directly in your AuraDB knowledge graphs. Aura Agent removes 
the complexity of integrating AI frameworks, models and infrastructure, so you 
can build, test, and deploy agents in minutes.




Key Features




 * Easily generate ontology-driven agents: Automatically generate agents 
tailored to your graph schema. With just a few details about your use case, you 
get a ready-to-test agent with instructions and prebuilt retrieval tools..
 * Higher accuracy with Agentic GraphRAG:  Combine vector search and graph 
queries for more precise retrieval, richer context and more accurate answers 
than traditional RAG approaches.
 * Transparent reasoning: Through detailed planning, tool execution and 
multi-hop graph data retrieval, agents provide clear and explainable answers.
 * One-click deployment: Deploy agents to a secure endpoint, ready for REST or 
MCP integration.

Preview-to-GA transition details


 Thank you to everyone who participated in Preview and helped shape Aura 
Agent. During Preview, the use of Aura Agent was entirely free.

As we move into General Availability, Aura Agent introduces usage-based 
pricing. Here’s how it works


Internal agents - No cost


 When agent visibility is set to Internal, you can create and test your agents 
within the Aura Console at no cost.


External agents - Usage-based pricing


 When agent visibility is set to External, your agent is accessible via the 
hosted Aura API or MCP endpoints and ready for integration into your AI 
applications and workflows. External agents are billed at $0.35 per agent-hour.


 * Grace period (through February 29, 2026):
All external agent usage remains free throughout February 2026, allowing teams 
to continue testing and validating their agents.
 * Billing starts March 1, 2026:
External agents will be billed at $0.35 per agent-hour. March usage will 
appear on your April 1, 2026 invoice.



 If you’d like to avoid usage charges, you can keep your agents internal and 
continue iterating on them in Aura Console at no cost.

 Read the Launch Blog  
<https://neo4j.com/blog/agentic-ai/neo4j-launches-aura-agent/>or explore the 
documentation <http://neo4j.com/docs/aura/aura-agent> to get started. 

We’re excited to see the agents you build with Aura Agent.

Hey there,



Here’s an insider tip: there are many ways to populate your graph database 
instance. A fuss-free approach is to useNeo4j Data Importer, a no-code tool to 
load data in your database.No complex query writing is required. 



To get started, watch this video on how to import a Kaggle Star Wars database 
into AuraDB:

 our console or bring your own to build the graph model you want. your content. 



 <https://console.neo4j.io/#create-database>

 * Username will be neo4j (no caps). 
 * Password is auto generated. Download the auto-generated password within the 
.txt file for every new database instance you create. In the file, you’ll find 
your database URL username, password, and instance name.

Alternatively, follow the step-by-step documentation 
<https://neo4j.com/docs/aura/auradb/getting-started/create-database/?utm_content=day3&utm_medium=email&utm_source=freeflow>
